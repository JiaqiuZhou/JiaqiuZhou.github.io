1. Complexity Analysis

1.1 Big-O Notation

  • Gives an upper bound of the complexity in the worst case, helping to quantify performance as the input size becomes arbitrarily large.
  • Complexitiex ordered in from smallest to largest; n - the size of the input
    • O(1)—Constant Time;O(log(n))—Logarithmic Time;O(n)—Linear Time;
    • O(nlog(n))—Linearithmic Time;O( )—Quadric Time;O( )—Cubic Time;
    • O( (b>1))—Exponential Time; O(n!)—Factorical Time

1.2 Big-O Properties

  • O(n+c)=O(n) O(cn) =O(n),c>0 O(f(n)) = O(f(n)中时间复杂度最高的那一个)

4.Stack

4.1 Discussion about Stacks

  • What is a Stack?

    • A stack is a one-ended linear data structure
    • Two primary operation Pop and push

  • When and where is a Stack used?

    • undo mechanisms(撤销机制) in text editors
    • matching brackets and braces(括号)
    • model a pile of books or plates(书堆模型)
    • support recursion by keeping track of previous 函数调用
    • Depth First Search(DFS) on a graph

  • Complexity Analysis

Operation Complexity
Pushing O(1)
Popping O(1)
Peeking O(1)
Searching O(n)
Size O(1)

  • Stack usage examples

    • []—>valid；{]}—>invalid

5.5 Removing elements from a BST

  • Find需要移除的节点，smaller move to left, larger move to the right

  • Remove我们想要移除的节点

    • Case1: 移除的节点没有子树，直接移除
    • Case2/3: 移除的节点有左子树或者右子树，直接移除接上即可
    • Case4: 移除的节点有左子树和右子树，找到右子树的数值最小的节点或者左子树数值最大的节点，然后移除节点，接上树
