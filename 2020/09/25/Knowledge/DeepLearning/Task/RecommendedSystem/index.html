介绍学习有关推荐系统得到的知识

1. Background

1.1 背景知识

  • 推荐系统: 我们希望给具有特定属性的用户推荐这一类用户所感兴趣的物品，对于系统来说，具有相同喜好的用户可以看成是一类相同的用户，推荐物品的属性需要符合这一类用户的喜好
  • 用户物品评分矩阵: 通过收集每一个用户对每一个物品的评分，我们就可以对用户物品矩阵进行构建，但是现实中我们不可能收集到所有的信息，因此这是一个稀疏矩阵
  • 用户表示与物品表示: 由上所述，具有相同物品评分的用户可以看成是一类，因此我们对于用户进行表示的话，可以用物品进行表示

1.2 任务定义

  • 任务定义: 其实对用户进行推荐物品，也就是对稀疏矩阵进行矩阵恢复的过程，然后从恢复后的矩阵中挑选出评分比较高的推荐给用户，就实现了推荐的过程
  • 难点与重点: 如何表示和利用用户特征和物品特征是这个任务的关键所在

1.3 数据集

1.3.1 Movielens1M

  • 细节可以见: https://github.com/chengstone/movie_recommender
  • 特点: 具有用户对电影的评分矩阵，用户和电影的各种属性，但是其中电影属性存在数据不齐全的情况
  • 用户属性: userID, 性别，年龄，职业
  • 电影属性: movieID, 电影名称，类型

2. 常见方法—协同过滤

2.1 矩阵分解SVD

  • 具体方法: 通过将用户物品评分矩阵进行分解，可以得到两个向量，通过对两个低维矩阵的操作，解决数据的稀疏问题

2.2 神经网络—NCF

  • 具体方法: 通过神经网络去模拟矩阵分解的过程，然后对用户和item的特征进行补充，重新组合成新的评分矩阵，利用神经网络来模拟矩阵组合的过程，并不断的通过SGD来优化模拟过程

3. 神经网络推荐系统

3.1 数据集测试方法—留一法

  • 数据形式: 多个用户的属性特征，多个物品的属性特征，用户对物品的评分矩阵
  • 数据切分: 把用户最新的数据留出一个出来作为测试集，然后剩余的数据作为训练集，对于剩余的数据，label就是训练集中用户对item的评分
  • 输入与输出: 输入用户特征与物品特征分别代表用户与物品，输出用户对这个物品的评分

3.2 重点与难点

  • 用户属性与item属性: 如何提取用户属性特征来表示用户，item属性特征表示item, 利用喜欢相同的item会是同一种用户这一点来对用户进行推荐类似的item
